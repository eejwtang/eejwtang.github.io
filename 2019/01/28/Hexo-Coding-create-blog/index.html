Hexo+Coding create blog

本文基于Hexo框架和Coding Pages服务搭建个人Blog。

1. 工具：

  • Hexo

    Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

  • Coding/GitHub Pages

    利用Coding/GitHub等git托管服务商提供的免费静态网站托管服务

3. 遇到的坑以及一些实用的功能

  • 将自己的域名映射到Pages上

    Blog页面搭建好了，就可以把自己的域名映射到Pages上了，我是在GoDaddy上购买的域名，用DNSPod做域名解析。

  • 创建不同的分支管理源码

    使用hexo d上传的是blog网页展示的部分，跟本地文件夹中的源码是不一样的，如果换台电脑，没有这个源文件夹就不能继续写作了。Git强大的版本控制系统提供了代码分支的技术，我们可以把源码换到和静态页面不同的分支，以解决这个问题。

    Coding的项目 Pages（即项目名称不为 <个性后缀>.coding.me）支持部署 coding-pages 分支和 master 分支。我们可以把刚刚的博客Pages部分的代码部署到coding-pages分支（在项目中创建该分支，并且在_config.yml中吧deploy中的branch改成coding-pages），空出来master默认分支来部署源码。

  • Git上传源码的时候遇到的坑

    前面折腾主题的时候，themes文件夹中会有很大git下来的文件夹，这就导致git文件夹中git文件的问题，如果强行push会导致子git文件夹变成灰色而不能管理，一番google，攻略如下Git管理Git子仓库

4. 9012年了为什么还要自己搭一个博客

最早在公众号上关注了一位大神GeekPlux，他在博客讲他的十年博客折腾历史，读来很受启发。自己搭建Blog，既Geek，又能帮助思考，还能分享知识，记录工作，emmmmm很棒。

新东方的艾力老师在讲34枚金币时间管理法的时候总会提到，没有记录就没有发生。人的大脑大概是一台复杂的高级处理器，但一定不是个可靠的信息存储设备，把最近折腾的东西记录下来，下次用到的时候可以随时回来翻，让大脑更专注于计算。

一年的小学教师的经历，我很了解，把自己掌握的知识拿出来给别人讲一遍，能更加深对知识的理解。

在互联网的海洋里找一个小角落，写点东西自娱自乐，反正这里没人跟你吵架。

还有这样可以督促我练习写作，我的研究生导师如是说：“研究生读到最后你会发现写代码是最快乐的事情，写论文一定会比写代码痛苦的。”
